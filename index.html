Understanding the Basics

What is Agentic AI?

The difference between asking for help and having a coding partner

Agentic AI goes beyond just answering questions. It can take actions, make decisions, and complete multi-step tasks while you stay in control.

Think of it this way: a regular AI assistant is like a textbook that answers questions. An AI agent is like a teaching assistant who can actually write code on your computer, run it, see errors, fix them, and keep going until the task is done.

Why this matters for beginners

  • You focus on the concepts while the agent handles the setup.
  • You learn a repeatable workflow: prompt ‚Üí review ‚Üí run ‚Üí refine.
  • You can stop, inspect, or undo before changes are made.
üí¨

AI Assistant

Great for explanations and quick snippets.

  • Answers questions
  • Writes code snippets
  • Does not edit your files
  • Does not run terminal commands
  • Cannot fix errors automatically

AI Agent

Built for real project work and iteration.

  • Answers questions and explains code
  • Edits your actual files with approval
  • Runs commands and tests
  • Sees errors and proposes fixes
  • Completes multi-step tasks

Now ask Claude to create the foundation for our Pac-Man game. Be specific.

Your Prompt

- An HTML file with a canvas element
- A black background with a simple maze made of blue walls
- A yellow Pac-Man circle that I can move with arrow keys
- Basic game loop with requestAnimationFrame

Example Fix Prompt

Troubleshooting
  • If Pac-Man gets stuck, check tile size and rounding.
  • If he still clips walls, make sure you test the next position, not the current one.

Time to add the iconic dots that Pac-Man eats and a score counter. You‚Äôll store dot data in the maze array and update it as Pac-Man collects dots.

Your Prompt
Add these features to the Pac-Man game:

1. Small white dots in all empty corridor spaces
2. When Pac-Man moves over a dot, it disappears
3. Each dot collected adds 10 points to the score
4. Display the score at the top of the screen

  • Draws dots and updates score in the game loop

Now for the challenge: add a ghost that chases Pac-Man using simple AI. At intersections, it chooses the path that gets closer to Pac-Man.

Your Prompt
Add a red ghost to the game:

1. Start the ghost in the center of the maze
2. The ghost should move automatically, trying to chase Pac-Man
3. Use simple AI: at each intersection, pick the direction that gets closest to Pac-Man
4. Ghost should respect walls (can't go through them)
5. Ghost should move slightly slower than Pac-Man so the player has a chance
6. If ghost touches Pac-Man, display "Game Over" and stop the game
üí° Pro Tip

If the game feels too hard or too easy, ask: ‚ÄúMake the ghost move 20% slower.‚Äù

Expected Agent Behavior
  • Adds a ghost entity with position + velocity
  • Implements intersection logic to choose directions
What You Should See

The ghost chases Pac-Man, respects walls, and the game stops on contact.

Troubleshooting
  • If the ghost jitters, ensure it‚Äôs snapping to tile centers.
  • If it gets stuck, add a fallback random direction at dead ends.

Implement the iconic power-up mechanic where Pac-Man can eat the ghost. This introduces a simple state machine for ghost behavior.

Your Prompt
Implement power pellet mechanics:

1. When Pac-Man eats a power pellet, start a 10-second "power mode"
2. During power mode, the ghost turns blue and moves away from Pac-Man
3. If Pac-Man touches the blue ghost, the ghost is "eaten" (200 points)
4. After being eaten, the ghost respawns in the center after 3 seconds
5. Make the ghost flash white during the last 3 seconds of power mode as a warning
6. Show a visual indicator (maybe a timer bar) for how long power mode lasts
üí° Pro Tip

If the ghost never returns to normal, ask the agent to walk through the timer logic.

Expected Agent Behavior
  • Adds a power mode timer and visual indicator
  • Changes ghost behavior based on state
What You Should See

The ghost turns blue in power mode, flees Pac-Man, and respawns after being eaten.

Troubleshooting
  • If power mode never ends, check your timer decrement.
  • If ghost never turns blue, verify you‚Äôre drawing based on state.

Add lives so players get multiple chances and a win condition when all dots are collected.

Your Prompt
Add lives and win condition:

1. Player starts with 3 lives, displayed as small Pac-Man icons
2. When ghost catches Pac-Man (not in power mode), lose 1 life instead of game over
3. After losing a life, reset positions but keep the score and remaining dots
4. Game over only when all lives are lost
5. When ALL dots and power pellets are collected, show "You Win!"
6. Add a "Press SPACE to restart" message after game over or win
üí° Pro Tip

Ask the agent to keep score and dots intact when resetting positions.

Expected Agent Behavior
  • Tracks lives and renders icons
  • Resets positions without resetting score/dots
What You Should See

Lives decrement on ghost contact, and a win screen appears once all dots are collected.

Troubleshooting
  • If lives never decrease, check collision logic for the ghost.
  • If win condition triggers early, verify you‚Äôre checking all dots and pellets.
